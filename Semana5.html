Lectura de datos de un archivo

Nos centraremos en el tipo de archivo de datos rectangular más común: CSV, “valores separados por comas”. Así es como se ve un archivo CSV simple.

Archivo CSV

  • La primera fila, comúnmente llamada fila de encabezado, proporciona los nombres de las columnas

  • Las siguientes seis filas proporcionan los datos.

  • Las columnas están separadas, es decir delimitadas, por comas.

  • Podemos leer este archivo en R usando read_csv().

  • El primer argumento es el más importante: la ruta al archivo.

Archivo CSV (II)

  • Puedes pensar en la ruta como la dirección donde se encuentra el archivo

  • El archivo se llama example_data.csv y se encuentra en la carpeta datos.

Consejos prácticos

Tratamiento de NA’s

  • Una vez que lee los datos, el primer paso generalmente implica transformarlos de alguna manera para que sea más fácil trabajar con ellos en el resto del análisis.

  • Echemos otro vistazo a los datos de ejemplo con eso en mente:

    • En la columna Sat hay un numero y luego la cadena de caracteres N/A, que debería haber sido un NA real que R reconocerá como “no disponible”.
    • Esto es algo que podemos abordar utilizando el argumento na.

Tratamiento de NA’s

  • De forma predeterminada, read_csv() reconoce solo cadenas vacías ("") en este conjunto de datos como NA;
  • Queremos que también reconozca la cadena de caracteres N/A

Otros argumentos de read_csv

Cadenas de texto con formato CSV

  • Hay un par de argumentos importantes más que debemos mencionar

  • Para eso usaremos datos de juguete

    Código 
    read_csv(
"a,b,c
1,2,3
4,5,6"
)
    # A tibble: 2 × 3
      a     b     c
  <dbl> <dbl> <dbl>
1     1     2     3
2     4     5     6

Otros argumentos de read_csv

  • Generalmente, read_csv() utiliza la primera línea de los datos para los nombres de las columnas

  • No es raro que se incluyan algunas líneas de metadatos en la parte superior del archivo.

  • Puede utilizar skip = n para omitir las primeras n líneas

  • También puede o utilizar comment = "#" para eliminar todas las líneas que comiencen con, por ejemplo, #

skip=n

Código 
read_csv(
"La primera fila de metadatos
La segunda fila de 
x,y,z
1,2,3
4,5,6",
skip = 2
)
# A tibble: 2 × 3
      x     y     z
  <dbl> <dbl> <dbl>
1     1     2     3
2     4     5     6

comment = "#"

Código 
read_csv(
"# Un comentario que deseo ignorar 
 # Otro comentario  
x,y,z
1,2,3
4,5,6",
comment = "#"
)
# A tibble: 2 × 3
      x     y     z
  <dbl> <dbl> <dbl>
1     1     2     3
2     4     5     6

col_names

  • Es posible que los datos no tengan nombres de columnas.

  • Puede usar col_names = FALSE para indicarle a read_csv() que no trate la primera fila como encabezado

    Código 
      read_csv(
  "1,2,3
  4,5,6",
  col_names=FALSE
  )
    # A tibble: 2 × 3
     X1    X2    X3
  <dbl> <dbl> <dbl>
1     1     2     3
2     4     5     6

col_names

  • Note que read_csv() coloca X_1, X_2, ... como nombres de columnas.

  • Alternativamente, puede pasar a col_names un vector de caracteres, que se utilizará como nombres de columnas:

    Código 
    read_csv(
"1,2,3
4,5,6",
col_names=c('x','y','z')  
)
    # A tibble: 2 × 3
      x     y     z
  <dbl> <dbl> <dbl>
1     1     2     3
2     4     5     6

Otros tipos de archivos

Otros tipos de archivos

Una vez que domine read_csv(), usar otras funciones read_ (lectoras) es sencillo; solo es cuestión de saber qué función usar.

  • read_csv2(): Lee archivos separados por punto y coma. Estos archivos usan ; en lugar de , para separar los campos y son comunes en países que usan , como separador decimal.

  • read_tsv(): Lee archivos delimitados por tabulaciones.

  • read_delim(): Lee archivos con cualquier delimitador e intenta adivinar automáticamente el delimitador si no se especifica.

  • read_fwf(): Lee archivos de ancho fijo. se puede especificar los campos por su ancho con fwf_widths() o por su posición con fwf_positions().

Otros tipos de archivos

  • read_table(): Lee una variación común de archivos de ancho fijo donde las columnas están separadas por espacios en blanco.

  • read_log() : Lee archivos de registro de estilo Apache.

Control de tipos de columnas

Control de tipos de columnas

  • Un archivo CSV no contiene información sobre el tipo de cada variable

  • Así que readr intentará adivinar el tipo. Estudiaremos:

    • cómo funciona el proceso de adivinación,
    • cómo resolver algunos problemas comunes que provocan fallos
    • cómo proporcionar los tipos de columna usted mismo.
    • daremos algunas estrategias generales útiles si readr presenta un fallo catastrófico y necesita comprender mejor la estructura de su archivo.

Adivinando los tipos de columnas

Para cada columna, extrae los valores de 1000 filas distribuidas uniformemente desde la primera hasta la última, ignorando los valores faltantes. A continuación, resuelve las siguientes preguntas:

  • ¿Contiene solo F, T, FALSE o TRUE (sin distinguir entre mayúsculas y minúsculas)? De ser así, es una columna lógica.
  • ¿Contiene solo números (p. ej., 1, -4,5, 5e6, Inf)? De ser así, es numérica.
  • ¿Cumple con el estándar ISO8601? De ser así, es una fecha o una fecha-hora.

Adivinando los tipos de columnas

  • De lo contrario, debe ser una columna de caracteres.

Cuidado

Esta heurística funciona bien si tienes un conjunto de datos limpio, pero en la vida real encontrarás una selección de fallas extrañas e interesantes.

Valores faltantes, tipos de columnas y problemas

  • La causa más común de fallo en la detección de columnas es que una columna contiene valores inesperados y se obtiene una columna de caracteres en lugar de un tipo más específico.

  • Una de las causas más comunes es la falta de un valor, registrado con un valor distinto del NA que readr espera.

  • Tomemos como ejemplo este sencillo archivo CSV de una sola columna:

Ejemplo

Código 
 simple_csv <- "
x
10
.
20
30"

Si lo leemos sin ningún argumento adicional, x se lee como una columna de caracteres:

Código 
read_csv(simple_csv) 
# A tibble: 4 × 1
  x    
  <chr>
1 10   
2 .    
3 20   
4 30

Cómo solucionarlo?

  • En este pequeño caso, se puede ver fácilmente el valor faltante.

  • ¿Qué ocurre si hay miles de filas con solo unos pocos valores faltantes representados por .’s dispersos entre ellas?

  • Una solución es indicar a readr que x es una columna numérica y luego ver dónde falla.

  • Esto se puede hacer con el argumento col_types, que toma una lista con nombre cuyos nombres coinciden con los de las columnas en el archivo CSV

Informando el tipo de columna

Código 
df <- read_csv(
simple_csv,
col_types = list(x = col_double())
)

Ahora read_csv() informa que hubo un problema y nos dice que podemos encontrar más información con problems()

Código 
problems(df)
# A tibble: 1 × 5
    row   col expected actual file                                              
  <int> <int> <chr>    <chr>  <chr>                                             
1     3     1 a double .      /private/var/folders/zg/q7h0m_912qzdnmx8cpymwp540…

Esto sugiere que este conjunto de datos usa . para los valores faltantes.

Solución na = "."

Entonces, establecemos na = ".", y la suposición automática funciona correctamente, obteniendo la columna numérica deseada.

Código 
read_csv(simple_csv,na = ".")
# A tibble: 4 × 1
      x
  <dbl>
1    10
2    NA
3    20
4    30

Tipos de columnas

readr proporciona un total de nueve tipos de columnas:

  • col_logical() y col_double() leen números lógicos y reales.

  • col_integer() lee enteros.

  • col_character() lee cadenas.

  • col_factor(), col_date() y col_datetime() crean factores, fechas y fechas y horas, respectivamente;

  • col_number() es un analizador numérico permisivo que ignorará los componentes no numéricos y es particularmente útil para las monedas.

Tipos de columnas

  • col_skip() omite una columna para que no se incluya en el resultado, lo que puede ser útil para acelerar la lectura de los datos si tiene un archivo CSV grande y desea utilizar solo algunas de las columnas.

Anular predeterminado

También es posible anular detección predeterminada del tipo de dato, cambiando de list() a cols() y especificando .default:

Código 
df2<- "
x,y,z
1,2,3"
read_csv(df2, col_types = cols(.default = col_character())
)
# A tibble: 1 × 3
  x     y     z    
  <chr> <chr> <chr>
1 1     2     3

Leer solo las columnas que especifique

Otra herramienta útil es cols_only(), que leerá únicamente las columnas que usted especifique:

Código 
read_csv(
df2,col_types = cols_only(x = col_character())
)
# A tibble: 1 × 1
  x    
  <chr>
1 1

Lectura de datos de varios archivos

  • A veces, los datos se dividen en varios archivos en lugar de estar en uno solo.
  • Por ejemplo, podrías tener datos de ventas de varios meses, con los datos de cada mes en un archivo separado:
    • 01-sales.csv para enero,

La columna file

  • El argumento id añade una nueva columna llamada file al marco de datos resultante, que identifica el archivo del que provienen los datos.

  • Esto es especialmente útil cuando los archivos que se leen no tienen una columna de identificación que permita rastrear las observaciones hasta sus fuentes originales.

  • Si tiene muchos archivos que desea leer, puede resultar complicado escribir sus nombres como una lista. En su lugar, puede usar la función list.files() para encontrar los archivos automáticamente mediante la búsqueda de un patrón en los nombres de archivo.

Escribir datos en un archivo

write_csv()

  • readr también incluye dos funciones útiles para escribir datos en el disco: write_csv() y write_tsv().

  • Los argumentos más importantes de estas funciones son x (el marco de datos que se guardará) y file (la ubicación donde se guardará).

  • También puedes especificar cómo se escriben los valores faltantes con na, así como si deseas añadir los datos a un archivo existente.

write_csv()

Código 
write_csv(ventas_data, "ventas.csv")
  • Cuando se configura una variable como factor, por ejemplo, esta información se pierde al guardar el archivo como csv y volver a leer nuevamente, ya que se lee de nuevo desde un archivo de texto sin formato.

write_rds()

write_rds() almacena los datos en el formato binario personalizado de R, llamado RDS. Esto significa que al recargar el objeto, con read_rds(), se carga exactamente el mismo objeto de R que se almacenó.

Código 
# para escribir
write_rds(ventas_data, "ventas.rds")
# para leer 
read_rds("ventas.rds")

Archivos parquet

El paquete arrow le permite leer y escribir archivos parquet, un formato de archivo binario rápido que se puede compartir entre lenguajes de programación.

Código 
library(arrow)
write_parquet(ventas_data, "ventas.parquet")
read_parquet("ventas.parquet")

parquet tiende a ser mucho más rápido que RDS y se puede utilizar fuera de R, pero requiere el paquete arrow.
