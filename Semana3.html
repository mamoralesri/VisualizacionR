Transformación de datos

Lectura de los datos

Ejercicio

  • Haga la lectura de los datos en el archivo acuicultura.csv
    • ¿Cuántas columnas contiene?
    • ¿Cuántas filas contiene?
    • ¿Cuántas piscinas se utilizaron en el estudio?
  • Las columnas OXIGESUP y OXIGEFON contienen medidas del oxígeno disuelto en la superficie y en el fondo de las piscinas, respectivamente. Escriba el código que permita realizar el siguiente gráfico, donde se comparan las distribuciones de oxígeno en superficie y fondo para cada piscina.

Ejercicios

Solución

Código 
acuicultura=read.csv2("data/acuicultura.csv")
# seleccionamos las columnas 
oxigeno=subset(acuicultura, select=c('PISCINA','dia','OXIGESUP','OXIGEFON')) 
oxigenol=melt(oxigeno,id.vars=c("PISCINA","dia"))
# cambiamos los nombres de columnas 
colnames(oxigenol)=c('Piscina','dia','Sitio','Oxigeno') 
# convertimos sitio en un factor 
oxigenol$Sitio=factor(oxigenol$Sitio,labels = c('Superf','Fondo'))
#head(oxigenol)
# piscina en un factor 
oxigenol$Piscina=factor(oxigenol$Piscina) 
#head(oxigenol)

tibbles

  • flights es un tibble, un tipo especial de marco de datos utilizado por tidyverse para evitar algunos problemas comunes.

dplyr

Lo básico de dplyr

  • Aprenderemos las funciones principales de dplyr, que nos permitirán resolver la gran mayoría de tus desafíos de manipulación de datos.

  • Antes de analizar sus diferencias individuales, vale la pena señalar lo que tienen en común.

    • El primer argumento es siempre un marco de datos.
    • Los argumentos subsiguientes normalmente describen en qué columnas operar, utilizando los nombres de las variables (sin comillas).
    • La salida es siempre un nuevo marco de datos.

Lo básico de dplyr. El pipe

  • Debido a que cada función hace solo una cosa, resolver problemas complejos generalmente requerirá combinar varias funciones, y lo haremos con el símbolo |>, conocido como pipe.

Ejemplo de pipe

Código 
res=flights |> filter(dest == "IAH") |> 
  group_by(year, month, day) |> summarize( 
    arr_delay  = mean(arr_delay, na.rm=TRUE)
    )

Tipos de funciones en dplyr

  • Las funciones de dplyr están organizadas en cuatro grupos según lo que operan:
    • filas,
    • columnas,
    • grupos y
    • tablas.
  • En las siguientes secciones, aprenderemos las funciones más importantes para filas, columnas y grupos
  • Luego volveremos a las funciones de unión que funcionan en las tablas.

Funciones de dplyr para filas

Las funciones más importantes de dplyr, que operan en las filas de un conjunto de datos son

  • filter(), que cambia las filas presentes sin cambiar su orden,

  • arrange(), que cambia el orden de las filas sin cambiar las que están presentes.

  • Ambas funciones afectan solo a las filas y las columnas no cambian.

Funciones de dplyr para filas

  • También analizaremos distinct(), que encuentra filas con valores únicos, pero a diferencia de arrange() y filter(), también puede modificar opcionalmente las columnas.

Operaciones sobre filas

filter()

  • filter() permite mantener filas en función de los valores de las columnas.
    • El primer argumento es el marco de datos.
    • El segundo argumento y los subsiguientes son las condiciones que deben ser verdaderas para mantener la fila.

  • Por ejemplo, podríamos encontrar todos los vuelos que salieron con más de 120 minutos (dos horas) de retraso

Operadores

  • > (mayor que)
  • >= (mayor o igual a)
  • < (menor que),
  • <= (menor o igual a),
  • == (igual a)
  • != (no igual a).

También puedes combinar condiciones con

  • & para indicar “y” (verifica ambas condiciones)
  • | para indicar “o” (verifica al menos una de las condiciones)

Vuelos de enero y febrero

Código 
flights |> filter(month == 1 | month == 2)

Cuando se combina | y == se puede usar %in%. En este caso mantiene las filas donde la variable es igual a uno de los valores de la derecha

Código 
flights |> filter(month %in% c(1,2))

Guardar el resultado de filter

  • Cuando dplyr ejecuta la operación de filtrado, crea un nuevo marco de datos y luego lo imprime.

  • No modifica el conjunto de datos original porque las funciones dplyr nunca modifican sus entradas.

  • Para guardar el resultado, debe utilizar el operador de asignación, <- o =

arrange()

  • arrange() cambia el orden de las filas según el valor de las columnas.

  • Se necesita un marco de datos y un conjunto de nombres de columnas (o expresiones más complicadas) para ordenar.

  • Si proporciona más de un nombre de columna, cada columna adicional se utilizará para desempatar los valores de las columnas anteriores.

arrange() , ejemplo

  • Por ejemplo, el siguiente código ordena por la hora de salida, que se distribuye en cuatro columnas. Primero obtenemos los primeros años, luego, dentro de un año, los primeros meses, etc.
Código 
flights|>arrange(year, month, day, dep_time)

Orden descendente

  • Puede utilizar desc() en una columna dentro de arrange() para reordenar el marco de datos según esa columna en orden descendente (de mayor a menor).

  • Por ejemplo, este código ordena los vuelos de mayor a menor retraso:

Código 
flights|>arrange(desc(dep_delay))

distinct()

Operaciones sobre columnas

Columnas

Hay cuatro funciones importantes que afectan a las columnas sin cambiar las filas:

  • mutate() crea nuevas columnas que se derivan de las columnas existentes,
  • select() cambia las columnas que están presentes,
  • rename() cambia los nombres de las columnas, y
  • relocate() cambia las posiciones de las columnas.

mutate()

  • El trabajo de mutate() es agregar nuevas columnas que se calculan a partir de las columnas existentes.
  • Más adelante aprenderemos un amplio conjunto de funciones que se pueden usar para manipular diferentes tipos de variables.
  • Por ahora, nos quedaremos con el álgebra básica, que nos permite calcular la ganancia, cuánto tiempo recuperó en el aire un vuelo retrasado y la velocidad en millas por hora:

Dónde colocar las nuevas columnas?

mutate() ¿qué columnas se conservan?

  • Se puede controlar qué variables se conservan con el argumento .keep .
  • Una opción particularmente útil para .keep es “used”, que especifica que conservamos solo las columnas que estuvieron involucradas o fueron creadas en el paso mutate().

¿qué columnas se conservan?

  • Por ejemplo, la salida del siguiente código contendrá solo las variables dep_delay, arr_delay, air_time, gain, hours y gain_per_hour:
Código 
flights |> mutate(
gain = dep_delay - arr_delay,
hours = air_time / 60,
gain_per_hour = gain / hours,
.keep = "used")

select()

  • No es raro obtener conjuntos de datos con cientos o incluso miles de variables. En esta situación, el primer desafío suele ser centrarse en las variables que nos interesan.

  • select() le permite seleccionar rápidamente un subconjunto útil utilizando operaciones basadas en los nombres de las variables:

Seleccionar columnas por nombre:

Código 
flights |> select(year, month, day)

Seleccionar todas las colummnas entre …

  • Seleccionar todas las columnas entre año y día (inclusive):
Código 
flights |> select(year:day)

  • Seleccionar todas las columnas excepto las de año a día (inclusive):
Código 
flights |> select(!year:day)

select(), continuación

  • Seleccionar todas las columnas que sean de caracteres:
Código 
flights|>select(where(is.character))

Seleccionar por nombre

  • Seleccionar todas las columnas cuyos nombres comienzan con “d”
Código 
flights|>select(starts_with("d"))

Seleccionar por nombre

  • Seleccionar todas las columnas cuyos nombres terminan en “time”
Código 
flights|>select(ends_with("time"))

Seleccionar por nombre

  • Seleccionar todas las columnas cuyos nombres contienen “time”
Código 
flights|>select(contains("time"))

Seleccionar por nombre

  • Todas las columnas cuyos nombres coinciden con x1, x2 y x3
Código 
flights|>select(num_range("x",1:3))

select(), continuación

  • Puedes cambiar el nombre de las variables a medida que las seleccionas utilizando =.
  • El nuevo nombre aparece en el lado izquierdo del =, y la variable antigua aparece en el lado derecho
Código 
flights |> select(tail_num = tailnum)

rename()

  • Si desea conservar todas las variables existentes y solo desea cambiar el nombre de algunas, puede usar rename() en lugar de select():
Código 
flights |> rename(tail_num = tailnum)
  • Si se quiere renombrar más de una columna se separan con coma.
Código 
flights |> rename(tail_num = tailnum,Anio=year)

relocate()

Utilice relocate() para mover las variables. Es posible que desees recopilar variables relacionadas o mover las variables importantes al frente. De forma predeterminada, relocate() mueve las variables al frente:

Código 
flights |> relocate(time_hour, air_time)

relocate() (cont)

También puedes especificar dónde colocarlos usando los argumentos .before y .after, tal como en mutate():

La tubería The Pipe

  • Hemos mostrado ejemplos sencillos del uso del pipe, pero su verdadero poder surge cuando se combinan varias funciones.

  • Por ejemplo, suponga que se quiere encontrar los vuelos mas rápidos al aeropuerto IAH de Houston: se requiere combinar filter(), mutate(), select() y arrange()

Código 
flights |>
filter(dest == "IAH") |>
mutate(speed = distance / air_time * 60) |>
select(year:day, dep_time, carrier, flight, speed) |>
arrange(desc(speed))

La tubería The Pipe (cont)

  • Aunque este proceso tiene cuatro pasos, es fácil de leer rápidamente porque las funciones aparecen al principio de cada línea: comenzar con los datos de los vuelos, luego filtrar, luego calcular (mutar), luego seleccionar y finalmente organizar.

La tubería The Pipe (cont)

  • ¿Qué pasaría si no tuviéramos la tubería?

  • Podríamos anidar cada llamada de función dentro de la llamada anterior:

Código 
arrange(
  select(
    mutate(
      filter(flights, dest == "IAH"),
    speed = distance / air_time * 60),
  year:day, dep_time, carrier, flight, speed
  ),
desc(speed) 
)

La tubería The Pipe (cont)

O podríamos utilizar un montón de objetos intermedios:

Código 
flights1<-filter(flights, dest == "IAH")
flights2<-mutate(flights1, speed = distance / air_time * 60)
flights3<-select(flights2,year:day,dep_time,carrier, flight,speed)
arrange(flights3, desc(speed))
  • Si bien ambas formas tienen su tiempo y lugar, la tubería generalmente produce un código de análisis de datos que es más fácil de escribir y leer.

group_by()

  • group_by() no cambia los datos, pero la salida indica que está “agrupada por” mes (Groups: month [12]).

  • Esto significa que las operaciones posteriores ahora funcionarán “por mes”.

  • group_by() agrega esta característica agrupada (denominada clase) al marco de datos, lo que cambia el comportamiento de las funciones aplicadas posteriormente a los datos.
